üöÄ End-to-End CI/CD Process (Simple & Clear)

This is the exact flow you should describe in an interview ‚Äî short, concrete, and repeatable.

1. Developer commits & pushes to GitHub

Typical commands used locally:
git add . git commit -m "update website" git push origin main

2. GitHub sends a webhook to Jenkins

Webhook target (Jenkins listens on 8080):
http://<jenkins-host>:8080/github-webhook/

3. Jenkins pipeline runs

Pipeline actions:
  • Checkout latest code from repository
  • Build Docker image from the project Dockerfile
  • Stop & remove previous container (if exists)
  • Run new container mapping host 8081 ‚Üí container 80
Jenkinsfile (copy/paste friendly):
pipeline { agent any stages { stage('Checkout') { steps { git branch: 'main', url: 'https://github.com/Yugendrarepo/my-sample-website' } } stage('Build Docker Image') { steps { sh 'docker build -t sample-website .' } } stage('Stop Old Container') { steps { sh 'docker stop sample-container || true' sh 'docker rm sample-container || true' } } stage('Run New Container') { steps { sh 'docker run -d --name sample-container -p 8081:80 sample-website' } } } post { success { echo 'Deployment successful ‚Äî site available on port 8081' } failure { echo 'Pipeline failed ‚Äî check logs' } } }

4. Dockerfile (project)

Minimal Dockerfile that serves your static site:
FROM nginx:alpine COPY index.html /usr/share/nginx/html/index.html

5. Ngrok for demo (public exposure)

Expose host port 8081 for public demo:
ngrok http 8081 # use the generated https://xxxx-xxxx.ngrok-free.dev URL to share

Quick verification

# Check container is running docker ps # View container logs docker logs -f sample-container # Verify locally curl http://localhost:8081

üõü Troubleshooting (common issues)
  • Webhook not firing: Check GitHub webhook delivery logs and Jenkins endpoint reachability (firewall/ngrok).
  • Jenkins can't run docker: Add Jenkins user to docker group or run docker with sudo.
  • Port conflict: Jenkins uses 8080, site uses 8081. Change mapping if needed.
  • Ngrok 502/404: Ensure the container responds locally at http://localhost:8081.
